用户：

- 用户应当支持通过邮箱注册和登录。
- 应当支持用户通过邮箱找回密码。
- 普通用户类型应当分为本班同学、校友和游客。本班同学会事先分发账号密码，其余所有人初始账号均为游客，游客可以通过向管理员提交相关证据升级为校友。游客不需要通过管理员审核。
- 用户应当可以修改个人信息，包括昵称、简介等。

帖子：

- 帖子的基本信息应当包括发布时间，内容缩略，浏览量，点赞数，评论数；点开后可以查看完整内容和评论列表。
- 应当仅支持本班同学和校友发布帖子，帖子可以包含文字、图片、视频。
- 应当支持所有用户浏览帖子、点赞、发布评论、给评论点赞和回复评论。
- 应当支持帖子匿名发布，匿名发布的帖子仅管理员可见发布者信息。
- 应当支持用户举报帖子和评论，管理员应当可以查看举报内容并进行处理。
- 用户应当可以设置帖子内容的可见范围，包括仅自己可见、本班同学可见、校友可见和所有人可见四种选项。

相册：

- **应当是瀑布流形式**
- **应当支持所有用户浏览相册图片和点赞。**

搜索：

- 包括帖子搜索和**相册搜索**（内容搜索，包括帖子的内容和相册的简介内容）

班日志：

- 游客不能浏览班日志
- **校友可以申请班日志查档时间，管理员审核通过后可以在约定的时间查看班日志内容**

管理员界面：

- **只有本班同学可以成为管理员。管理员可以由superuser任免，管理员的权限由superuser设置。**
- **superuser应当拥有所有管理员的权限**
- 管理员的所有可能被赋予的权限包括：
  - 班日志查档审批：审核校友提交的班日志查档请求
  - 禁言管理：禁言用户，解禁用户。
  - 用户权限管理：审核游客升级为校友的请求，禁言用户
  - 内容管理：审核举报内容
  - 相册管理：管理相册文件夹。
- **支持管理员向superuser申请修改权限，superuser应当审核这些请求**

superuser界面：

- **superuser可以分类别看到所有游客，校友，同学的账号；可以看到所有管理员的账号。**
- **支持superuser主动任免管理员，任免时可以选择管理员权限。**

通知（尚未和前端联调）：

- 应当支持实时通知
- 应当通知所有与当前用户有关的，由上级传达下的指令和事件，包括但不限于：
  - 管理员审核结果通知：当管理员审核用户的请求（如升级为校友、班日志查档请求等）后，应当通知用户审核结果。
  - 举报处理通知：当管理员处理用户举报的帖子或评论后，应当通知举报人处理结果。
  - 帖子互动通知：当用户发布的帖子被点赞、评论或回复时，应当通知该用户。
  - 系统公告通知：管理员发布系统公告时，应当通知所有用户。

**注：所有加粗的功能为尚未完成的功能**
