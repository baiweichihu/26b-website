用户：

- 用户应当支持通过邮箱注册和登录。
- 应当支持用户通过邮箱找回密码。
- 用户应当分为普通用户和superuser两种类型。
- superuser拥有任免管理员的权利，并且其自身也为管理员。
- 普通用户类型应当分为本班同学、校友和游客。本班同学会事先分发账号密码，其余所有人初始账号均为游客，游客可以通过向管理员提交相关证据升级为校友。游客不需要通过管理员审核，其余用户注册均需要管理员审核通过。
- 用户应当可以修改个人信息，包括昵称、简介等。

帖子：

- 帖子的基本信息应当包括发布时间，内容缩略，浏览量，点赞数，评论数；点开后可以查看完整内容和评论列表。
- 应当仅支持本班同学和校友发布帖子，帖子可以包含文字、图片、视频。
- 应当支持从班级相册中选择图片发布帖子。
- 应当支持所有用户浏览帖子、点赞、发布评论、给评论点赞和回复评论。
- 应当支持帖子匿名发布，匿名发布的帖子仅管理员可见发布者信息。
- 应当支持用户举报帖子和评论，管理员应当可以查看举报内容并进行处理。
- 用户应当可以设置帖子内容的可见范围，包括仅自己可见、本班同学可见、校友可见和所有人可见四种选项。

相册：

- 应当仅支持本班同学上传图片。
- 应当支持所有用户浏览相册图片和点赞。
- 本班同学和校友应当有申请删除相册图片的权限。
- 支持管理员对相册进行文件夹管理。

搜索：

- 包括帖子搜索和相册搜索（内容搜索，包括帖子的内容和相册的简介内容）

管理员界面：

- 管理员的所有可能被赋予的权限包括：
  - 班日志查档审批：审核校友提交的班日志查档请求
  - 用户权限管理：审核游客升级为校友的请求，禁言用户
  - 内容管理：审核举报内容
- 支持管理员向superuser申请修改权限，superuser应当审核这些请求

superuser界面：

- 支持superuser任免管理员，任免时可以选择管理员权限。
